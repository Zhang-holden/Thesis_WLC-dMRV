# 技术路线
## 论文题目

**"A Blockchain-Based Trusted Whole-Life Carbon Assetization Architecture for EV Charging Infrastructure"**

***

## 研究背景与痛点

- **地域**：新西兰
- <mark>**核心研究对象**：电动汽车 (EV) 充电基础设施</mark>
- **核心痛点**：
  - **物理边界缺失 (Physical Boundary Gap)**：现有的电动汽车充电系统 (Electric Vehicle Charging Systems) 碳积分方法学（如 Verra VM0038）在进行系统级评估时，物理边界极不完整。主流体系通常仅聚焦于运营阶段 (OpEx) 替代燃油车所产生的减排效益，却在很大程度上忽略了充电桩及配套硬件设备本身的隐含碳足迹。在长达 15-20 年的服役期中，建站期的 CapEx 隐含碳、B2-B5 阶段的维保换件排放、C1-C4 阶段的报废拆除、以及 D 阶段的回收循环经济效应均未被纳入统一的碳核算边界。充电桩从"出生"到"退役"的真实全生命周期碳账本从未真正闭合，这导致当前签发的碳信用存在被动高估的系统性缺陷。
  - **数字信任缺失 (Digital Trust Gap)**：碳排放数据的采集、计算和报告长期依赖中心化平台，不同利益相关方（运营商、电网公司、维保团队、审计机构）各自持有碎片化数据，缺乏一个跨组织的、不可篡改的、可独立验证的"唯一真相源 (Single Source of Truth)"。在此基础上发行的碳信用更面临"漂绿"质疑——减排量可能被重复计算，废弃阶段的环境负债无人担保，碳资产的底层数据质量难以让监管机构和碳市场买方信服。

***

## 研究问题 (Research Questions)

基于上述双重痛点，本研究提出以下两个研究问题：

- <mark>**RQ1 (全生命周期 dMRV 数据闭环的信度)**：能否通过区块链、物联网与动态 LCA 的融合，构建一个覆盖 EV 充电基础设施全生命周期 (Cradle-to-Grave) 的碳追踪数据闭环，并在数据完整性、链上可审计性与跨方可验证性三个维度上，提供比传统中心化 MRV 更强的可信基础？</mark>
- **RQ2 (机制层面的防漂绿闭环)**：能否通过智能合约自动化"碳债务初始化 → 净减排累积 → 碳偿付判定"的逻辑链，并借助分流与碳质押池(Carbon Escrow Vault) 前置锁定 EoL 阶段环境负债，从而从机制设计层面减少碳信用的漂绿风险？

***

## WLC-dMRV 四层架构设计

### Layer 1：物理资产与数据层 (Physical Asset & Data Layer)

**定位**：真实世界的投影，dMRV 中"MR"（监测与报告）的物理基础。

| 数据源              | 内容                                          | 对应 LCA 阶段                        |
| ---------------- | ------------------------------------------- | -------------------------------- |
| 充电站隐含碳数据         | 建站材料（钢材、混凝土、铜线缆等）+ 充电设备（充电桩、变压器、配电柜等）+ 施工安装 | A1-A5（CapEx / Balance of System） |
| 智能电表             | 实时高频充电数据 (kWh)                              | B6（OpEx / 转换与待机损耗）               |
| CMMS（计算机化维护管理系统） | 资产物理维保事件触发                                  | B2-B5（维修换件）                      |
| EoL 回收凭证         | 报废期真实回收记录                                   | C1-C4 / D（E-waste / 循环经济）        |

***

### Layer 2：预言机与整合层 (Oracle & Integration Layer)

**定位**：可信桥梁，防篡改地将链下数据传输上链。

| 数据类型 | 数据源                                                  | 传输方式                   |
| ---- | ---------------------------------------------------- | ---------------------- |
| 静态数据 | 上游制造商/供应商提供的产品碳足迹声明、国家级 EPD 数据库（如 BRANZ、Ecoinvent 等） | 去中心化预言机（如 Chainlink）上链 |
| 动态数据 | 新西兰 Transpower 实时边缘电网碳强度 MEF (gCO₂/kWh)，B2-B5阶段的维保事件 | API → 预言机 → 智能合约       |

***

### Layer 3：动态 LCA 与智能合约引擎层 (dLCA & Smart Contract Engine)

#### 模块 A：碳债务账本 (Carbon Debt Ledger)

```
Input: CapEx 隐含碳数据 + CMMS 维保换件信号

function initCarbonDebt(assetId, CapEx_embodied_carbon, BRANZ_EPD_data):
    ① 验证用户身份 (ClientIdentity)
    ② 检查资产是否已初始化（防重复）
    ③ 调用 Layer 2 预言机获取 BRANZ EPD 数据
    ④ 计算初始碳债务 = CapEx 材料量 × EPD 排放因子
    ⑤ 初始化红色负资产余额 (debt_balance = - carbon_debt)
    ⑥ 存入区块链状态数据库

function appendCarbonDebt(assetId, CMMS_event):
    ① Oracle 验证 CMMS 事件真实性
    ② 调用 EPD 数据计算替换件的隐含碳
    ③ 追加到 debt_balance（负资产增加）
    ④ emit 事件通知模块 C
```

#### 模块 B：动态运营 LCA 引擎 (Dynamic Operational LCA Engine)

```
Input: 智能电表高频充电数据 + Transpower MEF 实时电网碳强度

function calculateDailyNetReduction(assetId, charging_kWh, MEF):
    ① Oracle 获取 Transpower MEF（gCO₂/kWh）实时数据
    ② 计算实际充电碳排放 = charging_kWh × MEF
    ③ 获取燃油基准线排放 = charging_kWh × ICE_benchmark_factor
    ④ 计算每日净减排量 = 基准线排放 - 实际充电碳排放
    ⑤ 累计净减排量 += 每日净减排量
    ⑥ 更新状态，触发模块 C 的净正向判定
```

#### 模块 C：净正向触发器 (Net-Positive Tipping Point)

```
function checkNetPositive(assetId):
    ① 读取模块 B 的累计净减排量 (cumulative_reduction)
    ② 读取模块 A 的累计碳债务 (cumulative_debt)
    ③ 判定: net_position = cumulative_reduction + cumulative_debt

    if (net_position > 0):
        ④ 计算溢出量 = net_position
        ⑤ 调用 Layer 4 的分流阀门合约
        ⑥ Emit: "Carbon Net-Positive Achieved" 事件

    else:
        ⑦ 记录状态: "Still in Carbon Debt"
```

## LCA 评价模型构建

### 遵循标准

严格遵循 **ISO 14040** 与 **ISO 14044** 国际标准。

### 功能单位

<mark>1 kWh 的 EV 充电服务，或一个充电站全生命周期（如 20 年）。</mark>

### 系统边界

**全生命周期 (Cradle-to-Cradle / Cradle-to-Grave)**——这是相比现有研究（通常仅 Cradle-to-Gate）的显著进步：

| LCA 阶段 | 内容                      |
| ------ | ----------------------- |
| A1-A3  | 建材生产（钢材、混凝土、铜线缆、充电桩设备等） |
| A4-A5  | 运输与施工安装                 |
| B2-B5  | 维保替换（充电枪头、电缆、电力电子模块等）   |
| B6     | 运营电网能耗（动态 MEF 实时接入）     |
| C1-C4  | 报废拆除                    |
| D      | 回收与循环经济奖励               |

## 系统原型开发与 Web 界面

### 利益相关方与功能界面

| 利益相关方           | 功能界面                                 |
| --------------- | ------------------------------------ |
| **充电站运营商**      | 初始化资产（录入 CapEx 数据）、查看碳债务余额、查看每日净减排进度 |
| **维保团队**        | CMMS 事件上报接口、替换件隐含碳自动计算               |
| **电网数据提供方**     | Transpower MEF 数据预言机接入状态监控           |
| **碳信用买方 / 审计方** | 查看净正向碳信用生成记录、EoL 碳准备金池余额、可流通 VCUs 数量 |
| **回收方**         | EoL 回收凭证上传、触发准备金池解锁                  |

### 系统整体流程

```
初始化资产
  → CapEx 数据上链，模块 A 初始化碳债务
  → 每日 MEF 动态 LCA 运行，模块 B 计算净减排量
  → CMMS 维保事件触发，模块 A 追加碳债务
  → 模块 C 判定 net_position
  → 净正向：触发 Layer 4 分流铸造
  → EoL 阶段：回收凭证验证 → 准备金池解锁/销毁
```

## 核心创新点总结

1. **全生命周期系统边界**：从 Cradle-to-Gate 扩展到 Cradle-to-Cradle，特别是通过 Layer 4 碳准备金池机制首次将 EoL 废弃/回收阶段纳入碳资产化闭环
2. **动态电网碳因子**：通过 Transpower MEF API + 预言机实现每日碳强度的实时接入，而非使用静态年均排放因子
3. **三模块智能合约引擎**：碳债务账本 + 动态运营 LCA + 净正向触发器，实现全自动化的碳追踪与偿付判定
4. **防漂绿代币经济学**：分流阀门 + 强制 EoL 碳准备金池 + 真实回收凭证验证解锁，确保生成的碳信用无"漂绿"水分
5. **定制化数据库**：面向新西兰语境，集成 BRANZ EPD + Transpower MEF 两地缘数据库，提升对于新西兰资产管理的针对性和可靠性
